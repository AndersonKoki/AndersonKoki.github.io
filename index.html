Projects

Capacity Planning de Data Warehouse com Airbyte

O objetivo deste projeto é demonstrar como planejar os recursos computacionais necessários para implementar um Data Warehouse, seja local ou na nuvem. Fonte: Data Science Academy

Pipeline de Carga e Análise de Dados do Data Warehouse com Apache Airflow e Metabase

Construção de um pipeline de carga e análise de dados do Data Warehouse com Apache Airflow e Metabase. Depois de instalar o Apache Airflow e construir e configurar o DW, criar uma DAG no Airflow, usando Linguagem Python, para o processo de extração de dados no formato JSON, transformação dos dados de acordo com as regras de negócio e carga no DW. Com os dados carregados no DW, vamos extrair insights e realizar análise com o Metabase. Fonte: Data Science Academy

Azure Data Factory Para Pipeline de Carga de Dados no Azure SQL Database

Construção de um pipeline de dados usando o Azure Data Factory. Este pipeline de dados irá carregar os dados de um arquivo csv em um banco de dados SQL, tudo 100% na nuvem. Fonte: Data Science Academy

Pipeline de Analytics no Databricks data Lakehouse com delta Lake, Airbyte e Apache Spark

O objetivo do projeto é construir um Pipeline de Analytics que vai levar dados de um banco de dados relacional local para o processamento e análise em um Data Lakehouse na nuvem. Ferramentas como Docker, Airbyte e Apache Spark foram utilizados e o Data Lakehouse será construído com Databricks na nuvem AWS. Fonte: Data Science Academy

Contrução de Data Lake com MongoDB

O objetivo é construir um pipeline completo que será carregado em um Data Lake com MongoDB em Linguagem Python que inclui extração, transformação e carga de dados em um Data Lake. O desafio deste projeto é trabalhar com uma fonte de dados cuja estrutura não está bem definida, mas os dados precisam ser armazenados já em um formato adequado para o processo de análise, com alta performance. Fonte: Data Science Academy
